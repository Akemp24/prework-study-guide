HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user
  • DOCTYPE is a declaration that is not case sensitive
  • An HTML attribute defines additional information about an element
  • The head opening element contains the metadata for our web application.
  • Metadata is data about our webpage that thte browser uses to ser webpage to users. Not visisble by users
  • Title element defines the title for a webpage. Important for Search Engine Optimization or SEO.
  • The style element defines the CSS styles associated with a website
  • The base element define the base URL for a webpage
  • The link element connects an external resource to the HTML document
  • The meta element defines metadata such as the character set, description,keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The charset element sets our character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element set the default view for our webpage.
  • Elements that are nested inside other elements are called child elements. Parent element holds child elements.
  • ul, ol, li elements are unordered list, ordered lists, and list items.
  • img element contains information about images displayed on webpage.
  • br element creates a line of empty space, or a line break, between two blocks of contnet.
  • the a element stands for anchor and creates links to the same webpage or other webpages.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS aka cascading style sheet is used to define how webpage content is presented to users.It defines colors, aesthetics, visual layout, and organization.
  • The three ways to syle a webpage using CSS is Inline CSS, Internal CSS StyleSheet, and External CSS StyleSheet.
  • Inline CSS your styling element is directly in the HTML file. It is quick and permanent but not as flexible as external and internal CSS.
  • Internal CSS the rules for a webpage are in the head section of a HTML file. The rules only apply to that page and any change will apply to all taged page elements.
  • External CSS keeps the rules in a separate file. It it the most common because it makes design changes easier. Use a link element placed in the head to link the separate files.
  • Separation of concerns separates code into different files based on use.
  • A selector defines the element or attributes to which the rules, or declarations will apply.
  • Use an asterik symbol to apply a rule to all elements visible on the page.
  • Declarations: contain 2 important components; the css property we want to apply, and the value of the property.
  • Combine two selectors with a comma to allow the declarations to affect both selectors.
  • DRY: Do not Repeat Yourself
  • The img selector assigns property values to any image element in the markup
  • rel: specifies relationship betweeen current document and linked document resource.
  • href: specifies location URL of external resource.
  • Class attribute: share a css rule to any element we choose by assigning the rule to a class attribute with a class selector.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • make sure we are on the feature/add-html branch on from command line (git status)
  • to add the changed files use following command: git add -A
  • Now to commit the to our repository use this command: git commit -m "a summary/message of what you did"
  • git pull origin main: to pull base branch
  • git push origin feature/add-(name): to push up all changes
  • git status: to see what branch you are on
  • git checkout -b feature/add-"name" : to create a new branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • All strings are surrounded by quotation marks and each line of JS code ends with a semicolon.
  • console.log method to output a message to the webconsole by adding an argument.
  • link JS files to HTML files similar to how you would a CSS File.
  • The script.js is linked within the body element near the footer.
  • Use conditional statements and loops to control the order in which the code is executed.
  • When using if statment it will check to see if the condition is truthy. If it is then the action will execute if not then it will ignore.
  • When creating an if statement, we provide one or more conditions for the computer to evaluate.
  • An array is a group of related data.
  • The location of the data item in the array is identified with a unique number called and index.
  • First item in any array has a index of 0 not 1.
  • A for loop can tell the computer to go through each of the shapes in our array and log them to the console until there are no shapes left.
  • To find the length of the array use the built in property of an arrary named length.
  • A for loop contains three important statements: starting point for our loop, condition, and the final statement that allows the array to iterate over each item.
  • A function is simply a set of instructions that tells the computer how to perform a certain task.
  • Two requirements for a JavaScript function: Define the function, and call the function.
  • A function's name is followed by parentheses.
